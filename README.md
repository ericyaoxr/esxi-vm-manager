# ESXi 虚拟机管理器 - Web 版

一个基于 Web 页面的 ESXi 虚拟机管理工具，支持 Docker 部署，可在任何设备上通过浏览器管理你的虚拟机。

---

## 功能特点

- **图形化操作界面**：告别命令行，通过浏览器轻松管理虚拟机
- **批量操作**：勾选需要操作的虚拟机，一键挂起或启动
- **多服务器支持**：同时连接多台 ESXi 服务器，统一管理
- **收藏夹功能**：将常用的虚拟机组合保存为收藏，实现一键批量操作
- **资源监控**：查看服务器 CPU、内存、存储使用情况
- **定时间隔控制**：批量操作时可设置每台虚拟机的启动间隔时间

## 快速开始

### 第一步：安装 Docker

确认已安装 Docker 应用。

### 第二步：部署应用

#### 方法一：命令行部署

```bash
# 创建目录
mkdir -p /volume1/docker/esxi-vm-manager
cd /volume1/docker/esxi-vm-manager

# 登录镜像仓库（只需一次）
docker login ghcr.io -u ericyaoxr

# 拉取镜像
docker pull ghcr.io/ericyaoxr/esxi-vm-manager:latest

# 启动容器
docker run -d \
  --name esxi-vm-manager \
  -p 5000:5000 \
  -v $(pwd)/config:/app/config \
  -v $(pwd)/logs:/app/logs \
  --restart unless-stopped \
  ghcr.io/ericyaoxr/esxi-vm-manager:latest
```

#### 方法二：界面部署

1. 打开飞牛NAS **Docker 管理界面**
2. 点击 **镜像** → **拉取镜像**
3. 输入：`ghcr.io/ericyaoxr/esxi-vm-manager:latest` 或 `ghcr.milu.moe/ericyaoxr/esxi-vm-manager:latest`
4. 拉取完成后，点击 **容器** → **创建容器**
5. 配置参数：

| 配置项 | 值 |
|--------|-----|
| 容器名称 | esxi-vm-manager |
| 镜像 | ghcr.io/ericyaoxr/esxi-vm-manager:latest |
| 端口映射 | 5000:5000 |
| 卷挂载 | /volume1/docker/esxi-vm-manager/config → /app/config |
| 卷挂载 | /volume1/docker/esxi-vm-manager/logs → /app/logs |
| 重启策略 | unless-stopped |

### 第三步：访问使用

打开浏览器访问：`http://<服务器IP>:5000`

## 使用教程

### 1. 添加 ESXi 服务器

1. 点击顶部导航的 **连接配置**
2. 点击 **添加服务器**
3. 填写服务器信息：
   - **服务器名称**：给服务器起个名字（如"测试服务器"）
   - **ESXi 主机地址**：输入 IP 地址（如 192.168.1.100）
   - **用户名**：ESXi 用户名（通常是 root）
   - **密码**：ESXi 密码
4. 点击 **添加**

添加成功后，工具会自动测试连接是否正常。

### 2. 管理虚拟机

返回 **控制台** 页面，点击 **刷新** 按钮加载虚拟机列表。

#### 单个操作
- 点击虚拟机卡片上的 **挂起** 或 **启动** 按钮

#### 批量操作
1. 在 **虚拟机概览** 中勾选要操作的虚拟机（可多选）
2. 设置 **间隔时间**（批量操作时每台虚拟机之间的等待秒数）
3. 点击 **挂起选中虚拟机** 或 **启动选中虚拟机**

### 3. 使用收藏夹（快捷操作）

收藏夹可以帮你快速选中一组虚拟机，避免每次都要手动勾选。

#### 保存收藏
1. 在 **虚拟机概览** 中勾选需要的虚拟机
2. 点击 **保存选择**
3. 输入收藏名称（如"开发环境"、"测试服务器"）
4. 完成

#### 使用收藏
1. 在 **快捷收藏** 区域点击已保存的收藏名称
2. 该收藏包含的虚拟机会自动被选中
3. 直接点击 **挂起** 或 **启动** 按钮执行批量操作

#### 切换收藏
- 点击另一个收藏：自动切换到新的收藏选择
- 再次点击同一收藏：取消选择

### 4. 查看服务器状态

点击 **服务器** 标签，可以查看：

- **型号/厂商**：服务器的硬件信息
- **CPU**：使用率和核心数
- **内存**：已用/总量，使用率
- **存储**：每个数据存储的容量、剩余空间、使用率
- **虚拟机数量**：该服务器上的虚拟机总数

### 5. 查看日志

点击 **日志** 标签，查看系统操作记录。

## 高级功能

### API 接口

可通过 curl 命令调用：

```bash
# 获取虚拟机列表
curl http://localhost:5000/api/vms

# 获取服务器状态
curl http://localhost:5000/api/status

# 获取详细服务器信息
curl http://localhost:5000/api/servers/detail

# 启动指定虚拟机
curl -X POST http://localhost:5000/api/vm/start \
  -H "Content-Type: application/json" \
  -d '{"name": "VM名称", "server_host": "192.168.1.100"}'

# 挂起指定虚拟机
curl -X POST http://localhost:5000/api/vm/suspend \
  -H "Content-Type: application/json" \
  -d '{"name": "VM名称", "server_host": "192.168.1.100"}'
```

## 常见问题

### Q1: 连接 ESXi 失败？

1. 确认服务器地址、用户名、密码是否正确
2. 确认网络通畅（管理工具所在设备能访问 ESXi）
3. 确认 ESXi 已启用 SSH 或 API 访问

### Q2: 容器无法启动？

```bash
# 查看日志
docker logs esxi-vm-manager

# 检查容器状态
docker ps -a
```

### Q3: 如何更新到新版本？

```bash
# 拉取最新镜像
docker pull ghcr.io/ericyaoxr/esxi-vm-manager:latest

# 重启容器
docker restart esxi-vm-manager
```

### Q4: 收藏的虚拟机关机了还能用吗？

可以。收藏只是保存了虚拟机的名称和所属服务器，即使虚拟机关机，选择收藏时也会显示但会提示该虚拟机不可用。启动操作时只会对当前可用的虚拟机执行。
